Introduction

Welcome to the MPX Operating System User's Manual. This guide will help you understand how to use the MPX OS by explaining each command in a simple and straightforward manner. The User Interface works as follows. The main menu has 4 main options that a user can get to: Help which shows all commands, Time/Date Functions which includes all functions centered around the Time/Date as described below (Module 2), PCB functions which include any functions centered around manipulating PCBs as described below (Module 3), and finally the Version which shows the current version of this OS.

Commands

Help

Displays a list of all available commands and a brief description of each.

Version

Displays the current version of the MPX Operating System.

Time/Date Functions

Set Time

Sets the system time. Follow the prompts to enter the time.

First user will be prompted to enter the hours (Ex. 03).

Then user will be prompted to enter the minutes (Ex. 47).

Finally user will be prompted to enter the seconds (Ex. 50).

Get Time

Displays the current system time.

Set Date

Sets the system date. Follow the prompts to enter the date.

First user will be prompted to enter the month (Ex. 05).

Then user will be prompted to enter the day (Ex. 29).

Finally user will be prompted to enter the year (Ex. 03).

Get Date

Displays the current system date.

PCB Functions

Create PCB

Takes in user input of a name which must be under 10 characters and can't be a name that is already in use by another PCB. It then requires user input for the class which can either be a 0 or a 1 representing user (0) or system (1) processes. Finally it takes in user input for the PCB priority which can be any integer 0-9 with 0 being the highest and 9 being the lowest. This method uses that information to create a PCB with the specified attributes.

Delete PCB

This function takes in user input of the name of a PCB to be deleted. The name must be one that exists as a PCB in memory to work properly (elsewise the function will return to the menu screen). Once a valid name is provided the PCB named will be deleted.

Block PCB

This function takes in a user input of a name of a PCB to be moved to the blocked state. The name entered must be one that corresponds to a PCB that is currently being stored in memory. The PCB specified must also not be a system process for this to work and it must not already be in the blocked state. If these conditions are true the PCB will be moved from the ready queue into the blocked queue.

Unblock PCB

This function takes in a user input of a name of a PCB to be moved to the unblocked/ready state. The name entered must be one that corresponds to a PCB that is currently being stored in memory. It also must not already be in the unblocked/ready state, if both of those conditions are true the PCB will be moved from the ready queue into the blocked queue.

Suspend PCB

This function takes in a user input of a name of a PCB to be moved to a suspended state. The name entered must be one that corresponds to a PCB that is currently being stored in memory. The PCB specified must also not be a system process for this to work. If both of those conditions are true the PCB will be moved from into the suspended state.

Resume PCB

This function takes in a user input of a name of a PCB to be moved out of the suspended state. The name entered must be one that corresponds to a PCB that is currently being stored in memory. The PCB specified must also be in the suspended state. If both of those conditions are true the PCB will be moved out of the suspended state.

Set Priority

This function takes in a user input of a name and a priority value to change the priority of an already created PCB. The name entered must be one that corresponds to a PCB that is currently being stored in memory. The priority entered must also be a valid priority value of between 0 and 9. If both of those conditions are true the PCB will be updated to represent the new priority given by the user.

Show PCB

This function takes in a user input of a name of a PCB to be displayed. The name entered must be one that corresponds to a PCB that is currently being stored in memory. The PCB specified must will be displayed to the user along with its attributes.

Show Ready

This function takes in no user input. It will go through the ready queue and display to the user all of the current PCBs stored in the ready queue.

Show Blocked

This function takes in no user input. It will go through the blocked queue and display to the user all of the current PCBs stored in the blocked queue.

Show All

This function takes in no user input. It will go through the ready queue and the blocked queue and display to the user all of the current PCBs stored in the ready and blocked queues.

Shutdown

To safely shut down the MPX Operating System, type the word "shutdown". You will be asked to confirm your choice. Type 'y' to confirm and proceed with the shutdown. 'n' will cancel the shutdown process
